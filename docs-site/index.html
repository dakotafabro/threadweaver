ThreadWeaver

Consent-first conversational memory for Goose sessions, designed for strategic continuity and trust-preserving retrieval.

Problem

"I do a lot of thinking in my ChatGPT app and I was tired of copying and pasting my chats and/or asking ChatGPT to extract the chat into an .md file. With that, ThreadWeaver was born. I can upload a backlog of my chat history via ChatGPT export (settings in your ChatGPT profile) and also pull in the latest chats as well. This lets me reference prior thinking from my chat history and integrate it with what I'm building with Goose."

Installation

git clone https://github.com/dakotafabro/threadweaver.git
cd threadweaver
npm install
npm run -w @threadweaver/cli dev -- init

Use in a Goose Session

I remember thinking in ChatGPT about X.
Surface that and connect it to our build on Y.

ThreadWeaver resolves intent, applies policy scope, retrieves relevant prior thinking, and returns strategic synthesis.

ChatGPT Export Setup

  1. Request your ChatGPT export data. This may take several hours or days depending on export size.
  2. When you receive the email that your export is ready, ensure you are logged into the ChatGPT account associated with that export.
  3. Download the export archive from the email link.
  4. Unzip the export archive locally.
  5. Provide the path to the unzipped export directory to Goose to configure ThreadWeaver.
threadweaver connect chatgpt-export --file /path/to/unzipped-export-directory
threadweaver projects sync-from-import
threadweaver projects allow --project chatgpt-general --access summary

Design Decisions

  • Local-first for trust, privacy, and velocity.
  • Consent-first with deny-by-default project scope controls.
  • Deterministic policy enforcement and auditability.

Alignment

Goose doubles down on open in latest two releases

ThreadWeaver aligns through SDK/MCP boundaries, policy-governed retrieval, and an open integration posture.
