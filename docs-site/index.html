ThreadWeaver

Consent-first conversational memory for Goose sessions, designed to preserve strategic continuity across long-horizon build workflows.

The Problem

“I do a lot of thinking in my ChatGPT app and I was tired of copying and pasting my chats and/or asking ChatGPT to extract the chat into an .md file. With that, ThreadWeaver was born. I can upload a backlog of my chat history via ChatGPT export (settings in your ChatGPT profile). This lets me reference prior thinking from my chat history and integrate it with what I’m building with Goose.”

Goose-First Usage

In a Goose session, ask naturally:

I remember thinking in ChatGPT about X.
Surface my thought and connect it to what we're building on project Y.

ThreadWeaver infers intent, applies policy-scoped retrieval over ingested export data, and returns strategic synthesis.

Quickstart

git clone https://github.com/dakotafabro/threadweaver.git
cd threadweaver
npm install
npm run -w ./cli dev -- init

goose plugin install file:///absolute/path/to/threadweaver

threadweaver connect chatgpt-export --file /absolute/path/to/export-or-conversations.json
threadweaver projects sync-from-import
threadweaver projects allow --project chatgpt-general --access summary

threadweaver projects list
threadweaver recent --project chatgpt-general --limit 10
threadweaver recall "query text" --project chatgpt-general --limit 5

ChatGPT Export Setup (Step-by-Step)

  1. Request your ChatGPT export data. Depending on size, this may take several hours or days.
  2. When your export email arrives, ensure you are logged in to the ChatGPT account associated with that export.
  3. Download the export archive.
  4. Unzip the export archive on your local machine.
  5. Provide the path to the unzipped export directory to Goose to configure ThreadWeaver.
threadweaver connect chatgpt-export --file /path/to/unzipped-export
threadweaver projects sync-from-import
threadweaver projects allow --project chatgpt-general --access summary

Design Decisions

  • Local-first: keep memory local by default for trust and fast iteration.
  • Consent-first: deny-by-default project scope controls.
  • Deterministic security: policy enforcement remains strict regardless of parser path.
  • Auditability: retrieval and scope decisions are logged for transparency.
