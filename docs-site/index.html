ThreadWeaver

Consent-first conversational memory for Goose sessions, with policy-scoped retrieval and strategic synthesis.

Problem

I do a lot of thinking in my ChatGPT app and I was tired of copying and pasting my chats and/or asking ChatGPT to extract the chat into an .md file. With that, ThreadWeaver was born. I can upload a backlog of my chat history via ChatGPT export (settings in your ChatGPT profile) and also pull in the latest chats as well. This lets me reference prior thinking from my chat history and integrate it with what I'm building with Goose.

Installation

git clone https://github.com/dakotafabro/threadweaver.git
cd threadweaver
npm install
npm run -w @threadweaver/cli dev -- init

Use in a Goose session

ThreadWeaver is intended to work with Goose first, while maturing in portability.

I remember thinking in ChatGPT about X.
Surface it and show overlap with what we are building for project Y.

ChatGPT Export Setup (Step-by-Step)

  1. Request your ChatGPT export data. This may take several hours or days depending on export size.
  2. When you receive the email that your export is ready, ensure you are logged in to the ChatGPT account associated with that export.
  3. Download the export archive from the email link.
  4. Unzip the export archive locally.
  5. Provide the path to the unzipped export directory to Goose to configure with ThreadWeaver.
threadweaver connect chatgpt-export --file /path/to/unzipped-export-directory
threadweaver projects sync-from-import
threadweaver projects allow --project chatgpt-general --access summary

Design decisions

  • Local-first: lower privacy risk and fast iteration.
  • Consent-first: deny-by-default and project-scoped access.
  • Deterministic enforcement: policy checks remain strict even with flexible intent parsing.
  • Auditability: retrieval decisions are logged.

Goose ecosystem alignment

ThreadWeaver aligns with updates described by Angie Jones: Goose doubles down on open in latest two releases.

  • Runtime/interface separation
  • Policy-governed retrieval and tool contracts
  • Portable integration approach for broader agent ecosystems
